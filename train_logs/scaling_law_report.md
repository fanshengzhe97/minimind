# MiniMind Chinchilla-Style Scaling Law 拟合报告

## 概述

本项目对 MiniMind 系列预训练模型（6个不同大小）的训练数据进行 Chinchilla 风格的缩放律（Scaling Law）拟合。缩放律描述了模型损失 $L$ 与模型参数量 $N$ 和训练数据量 $D$ 之间的幂律关系。

### 核心公式

$$L(N, D) = E + \frac{A}{N^\alpha} + \frac{B}{D^\beta}$$

其中：
- $N$ — 模型参数量（个）
- $D$ — 训练 token 数（个）
- $L$ — 交叉熵损失（Cross-Entropy Loss）
- $E$ — 不可约损失（Irreducible Loss），即理论最小损失
- $A, \alpha$ — 参数项系数和指数
- $B, \beta$ — 数据项系数和指数

## 方法：三种拟合方式（Chinchilla 原文 Approach 1-3）

### 关于 α=β 约束的重要说明

在 Chinchilla 论文中，$\alpha \approx \beta$，因此最优 D/N 比（token/参数比）为常数——这是 scaling law 最基本的物理意义之一。

本报告的初始无约束拟合得到 $\alpha=0.332$、$\beta=0.654$（$\beta \gg \alpha$），导致 D/N 比随模型大小变化，这**不合理**。原因是：
1. 只有 6 个模型，$N$ 仅跨越约 1 个数量级
2. 所有模型在同一训练步看到相同的 $D$
3. $\alpha$ 和 $\beta$ 高度相关，数据不足以可靠分离

**解决方案**：约束 $\alpha = \beta$ 进行拟合，$R^2$ 仅下降 0.01，但得到物理意义更合理的常数 D/N 比。

因此**最终推荐结果采用约束 $\alpha=\beta$ 的拟合**。

### 损失函数

三种方法均使用 **log-space Huber loss**：

$$\mathcal{L}_{\text{Huber}} = \sum \text{Huber}(\delta=10^{-3},\ \log L_{\text{pred}} - \log L_{\text{obs}})$$

即对**相对误差**做 Huber 损失，确保 loss 值较大和较小时误差权重均衡。

### Method 1：联合参数拟合（Parametric Fit）

**原理**：直接对所有 762 个数据点 $(N_i, D_i, L_i)$ 拟合完整公式。

**步骤**：
1. 200 次随机初始值搜索
2. 分别用 L-BFGS-B 和 TNC 两个优化器
3. 选最优结果（最小 Huber loss）
4. 500 次 Bootstrap 计算 90% 置信区间

**结果（无约束）**：

| 参数 | 拟合值 | 90% 置信区间 |
|:--:|:--:|:--:|
| $E$ | 1.561 | [1.538, 1.584] |
| $A$ | 275.30 | — |
| $\alpha$ | 0.332 | [0.331, 0.334] |
| $B$ | 327142 | — |
| $\beta$ | 0.654 | [0.651, 0.655] |
| $R^2$ | **0.977** | — |

**结果（约束 $\alpha=\beta$，推荐）**：

| 参数 | 拟合值 | 90% 置信区间 |
|:--:|:--:|:--:|
| $E$ | 1.772 | [1.755, 1.786] |
| $A$ | 25742 | — |
| $\alpha=\beta$ | 0.609 | [0.608, 0.610] |
| $B$ | 143085 | — |
| $R^2$ | **0.967** | — |
| D/N | **16.7** | [16.6, 16.8] |

### Method 2：IsoFLOP 分析

**原理**：对每个 FLOP 预算（即每个训练步），找到使 loss 最小的模型大小 $N_{\text{opt}}$，然后拟合 $N_{\text{opt}} \propto C^a$。

**步骤**：
1. 每个训练步构成一个 IsoFLOP 轮廓
2. 对每个轮廓（6个模型），用三次样条插值找到最优 $N$
3. RANSAC 鲁棒拟合 $N_{\text{opt}} = k \cdot C^a$
4. 只使用后半段训练数据（loss 趋于稳定后）

**发现**：在所有 FLOP 预算下，最大模型（481.7M）始终最优。

**解释**：当前模型的训练 token 相对较少（约 4.4B），远未达到 Chinchilla 最优分配点（通常 token/参数 ≈ 20:1）。对于 481.7M 模型，最优 token 量约为 9.6B，而实际只训练了 4.4B。因此在当前规模下，**增加模型参数比增加训练数据更有效**。

### Method 3：分离拟合 + 交叉验证

**原理**：两步分离拟合，再通过 k-fold 交叉验证评估泛化性能。

**步骤**：
1. **第一步**：对每个模型（固定 $N$），拟合 $L(D) = E_N + B/D^\beta$（共享 $B, \beta$）
2. **第二步**：对得到的 $E_N$ 值，拟合 $E_N = E + A/N^\alpha$
3. **5-fold 交叉验证**：在训练集上优化完整模型，在验证集上评估

**结果**：

| 参数 | 拟合值 |
|:--:|:--:|
| $E$ | 1.360 |
| $A$ | 71.48 |
| $\alpha$ | 0.243 |
| $B$ | 406068 |
| $\beta$ | 0.664 |
| $R^2$ (原始空间) | **0.980** |
| 5-fold CV $R^2$ | **0.980 ± 0.002** |

交叉验证标准差仅 0.002，说明模型泛化能力优秀。

## 最优计算分配

### 理论基础

给定计算预算 $C \approx 6ND$（前向+反向传播的近似 FLOPs），当 $\alpha=\beta$ 时，最优分配满足：

$$D_{\text{opt}} = N_{\text{opt}} \cdot r, \quad r = \left(\frac{B}{A}\right)^{1/\alpha}$$

最优参数量和 token 量为：

$$N_{\text{opt}} = \sqrt{\frac{C}{6r}}, \quad D_{\text{opt}} = \sqrt{\frac{Cr}{6}}$$

即 $N_{\text{opt}} \propto C^{0.5}$, $D_{\text{opt}} \propto C^{0.5}$。

### 约束 $\alpha=\beta$ 结果

$$r = \frac{D_{\text{opt}}}{N_{\text{opt}}} = \left(\frac{143085}{25742}\right)^{1/0.609} \approx 16.7$$

| C (FLOPs) | $N_{\text{opt}}$ | $D_{\text{opt}}$ | $D/N$ |
|:--:|:--:|:--:|:--:|
| $10^{18}$ | 100M | 1.67B | 16.7 |
| $2\times 10^{18}$ | 141M | 2.36B | 16.7 |
| $5\times 10^{18}$ | 223M | 3.73B | 16.7 |
| $10^{19}$ | 316M | 5.28B | 16.7 |
| $2\times 10^{19}$ | 447M | 7.47B | 16.7 |
| $5\times 10^{19}$ | 706M | 11.8B | 16.7 |
| $10^{20}$ | 998M | 16.7B | 16.7 |

## 附录：评估指标说明

### $R^2$（决定系数）

$R^2$ 衡量拟合公式能解释多少数据中的变化，计算方式为：

$$R^2 = 1 - \frac{\sum_i (L_i - \hat{L}_i)^2}{\sum_i (L_i - \bar{L})^2}$$

- $L_i$ — 第 $i$ 个数据点的实际 loss
- $\hat{L}_i$ — 用拟合公式预测的 loss
- $\bar{L}$ — 所有实际 loss 的平均值

**直观理解**：

| $R^2$ | 含义 |
|:--:|:--|
| 1.0 | 完美拟合，预测完全准确 |
| 0.97 | 公式解释了 97% 的 loss 变化 |
| 0.0 | 拟合不比直接用平均值好 |
| $<0$ | 拟合比直接用平均值更差 |

**代码实现**（`fit_scaling_law.py` 第 169 行）：

```python
r2 = 1 - np.sum((L - L_pred)**2) / np.sum((L - np.mean(L))**2)
```

分子 $\sum (L - \hat{L})^2$ 是**残差平方和**，即公式没解释的部分；分母 $\sum (L - \bar{L})^2$ 是**总平方和**，即数据自身的总波动。

**为什么还算 log 空间的 $R^2$？**

因为 loss 值从 ~6.6（初始）降到 ~2.0（收敛），跨度很大。如果只在原始空间算 $R^2$，loss 大的点会主导结果。log 空间的 $R^2$ 计算方式相同，只是把 $L$ 换成 $\log L$：

$$R^2_{\log} = 1 - \frac{\sum (\log L_i - \log \hat{L}_i)^2}{\sum (\log L_i - \overline{\log L})^2}$$

这相当于用**相对误差**衡量拟合质量。两种 $R^2$ 都 >0.96，说明拟合在高 loss 和低 loss 区域都同样好。

### Bootstrap 置信区间

由于拟合参数 $E, A, \alpha, B, \beta$ 是优化结果，存在统计不确定性。我们使用 Bootstrap 方法（有放回重采样 500 次）来估计参数的可信范围：

1. 从 762 个数据点中有放回地抽取 762 个点（称为一个 Bootstrap 样本）
2. 对这个样本重新拟合参数
3. 重复 500 次，得到 500 组参数估计
4. 取 5% 和 95% 分位数作为 90% 置信区间

例如 $\alpha = 0.609$，90% 置信区间 [0.608, 0.610] 意味着：我们有 90% 的信心 $\alpha$ 的真实值在 0.608 到 0.610 之间。

### 5-fold 交叉验证（Method 3）

将 762 个数据点随机分成 5 份，每次用 4 份训练、1 份验证，轮换 5 次。$R^2=0.980 \pm 0.002$ 说明拟合的泛化性能非常稳定：无论用哪份数据验证，$R^2$ 波动仅 0.002。

### 与 Chinchilla 论文对比

| 指标 | Chinchilla (Hoffmann 2022) | MiniMind（推荐） |
|:--:|:--:|:--:|
| $\alpha$ | 0.34 | **0.61** |
| $\beta$ | 0.28 | **0.61** |
| 是否约束 $\alpha=\beta$ | 否（但 $\alpha \approx \beta$） | **是（数据不足以分离）** |
| $N_{\text{opt}} \propto C^a$ | $a \approx 0.46$ | $a = 0.50$（因 $\alpha=\beta$） |
| 最优 D/N 比 | $\sim 20:1$ | **16.7:1** |

**差异解释**：
- MiniMind 的 $\alpha=\beta=0.61$ 大于 Chinchilla 的 0.34/0.28，说明损失下降更快（模型规模小，处于 scaling law 的早期阶段）
- D/N=16.7 略低于 Chinchilla 的 20，说明小规模模型每个参数需要的 token 略少
- 两者在数量级上一致，验证了缩放律的普适性

## 结论

1. **约束 $\alpha=\beta$ 是最合理的选择**：无约束拟合下 $\alpha$ 和 $\beta$ 无法可靠分离（数据有限），约束后 $R^2$ 仅降 0.01
2. **MiniMind 缩放律**：$L(N, D) = 1.772 + 25742/N^{0.609} + 143085/D^{0.609}$，$R^2=0.967$
3. **常数 D/N 比**：每个参数最优对应 **16.7 个 token**（90% CI: [16.6, 16.8]），与 Chinchilla 的 ~20 接近
4. **对 481.7M 模型**：当前 3.45B tokens 不足，最优需要 **8.0B tokens**（还需 2.3 倍）
5. **对 206.5M 模型**：当前训练量恰好最优
6. **小模型（53.3M）**：已严重数据过剩（D/N=64.8），继续训练收益极低
